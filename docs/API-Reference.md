# API Reference

If your mod wants to go further and interact deeply with the engine, Ores Core provides a large, flexible Java API. All methods are accessible via the `ores.mathieu.api.OresCoreAPI` class.

> 💡 **Tip:** If you feel a specific API function would be relevant or helpful for your own mod, please don't hesitate to **[propose a new feature](https://github.com/MathieuDorval/ORES-CORE/issues/new?template=feature_request.md)**! We are always looking to expand the API to meet community needs.

## 🔄 Reloading
- `void reloadRegistry()`: Clears the current registry data and triggers a full scan of `registry.json` files.
- `void reloadVeinGeneration()`: Reloads the ore generation configs from `ores-generation.toml` without a full restart.
- `void registerResources()`: Triggers the Minecraft Registry phase for newly added dynamic IDs.

## 🆔 ID Queries
- `boolean canCreateId(String id)`: Checks if a specific ID can theoretically be generated by ORES CORE.
- `Map<String, String> getCreatableIds()`: Returns the complete list of objects ORES CORE can create, including aliases.
- `boolean isIdAdded(String id)`: Checks if an ID is active in the current registry.
- `Set<String> getCurrentlyAddedIds()`: Returns all active IDs created by the core.
- `Set<String> getCurrentlyAddedHostBlocks()` / `getModProvidedHostBlocks()` / `getConfigHostBlocks()`: Manage and query active Host Stones.
- `Set<String> getCurrentlyAddedOreMaterials()`: Returns materials currently linked to dynamic ore generation.

## 💉 API Overrides
- `injectPropertyOverride(String category, String id, String property, String value)`: Injects an in-memory property override (lower priority than user TOML configs).
- `getApiMaterialOverrides()`, `getApiItemOverrides()`, `getApiBlockOverrides()`: Returns maps of the currently injected overrides.

## 📊 Properties Queries
- `String getMaterialProperty(String materialName, String property)`: Retrieves a specific property for a material (e.g., `hardness`, `resistance`).
- `Map<String, String> getAllMaterialProperties(String materialName)`: All properties of a material.
- `getItemProperty` / `getBlockProperty` / `getChemicalProperty`: Same queries tailored for specific instances.

## 🏷️ Tags & Blacklist
- `Set<String> getTags(String id)`: Dynamically calculates Minecraft tags for any item/block handled by Ores Core.
- `boolean isBlacklisted(String id)`: Checks if an ID is ignored based on user settings.
- `Set<String> getBlacklistedIds()`: Gets all ignored IDs.

## 🌍 World Generation
- `boolean isMaterialGenerated(String material)`: Checks if a material is set to naturally generate as an ore.
- `List<VeinConfig> getVeinGenerationProperties(String material)`: Retrieves the list of generation properties tailored to a specific material.
