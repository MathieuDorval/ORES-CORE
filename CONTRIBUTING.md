# 🤝 Contributing Guide: ORES-CORE

Thank you for your interest and welcome! 
If you wish to propose new additions or changes to *ORES-CORE*, this guide is for you. I have set up a structure to facilitate your proposals.

**Important Note:** Everyone is definitely welcome to propose modifications, additions, or improvements! However, please be aware that all proposals are subject to careful review, and **only the repository owner** can validate, approve, or deny them.

---

## 📌 1. How to propose your modifications?

You can contribute in two main ways on GitHub:

1. **If you are not familiar with coding or Pull Requests:**
   You can use the **Issues** tab > **New Issue**. 
   I have prepared a specialized form ("*🛠️ Content Addition/Modification Proposal*") that will ask you for the category of your addition, your code proposal, and your explanations.

2. **If you are a developer and know Git (Pull Request):**
   Create a branch, modify what you want, and open a *Pull Request*. 
   A pre-filled template will appear to help you structure your title (e.g., `[Material] Modify Zinc`) and your explanations correctly.

---

## 🛠️ 2. Which files to modify depending on the addition?

The key locations where the magic happens are mainly found in the `common/src/main/java/ores/mathieu/` folder.

### 🪨 Materials / Base
- **File:** `common/src/main/java/ores/mathieu/material/MaterialsDatabase.java`
- **Action:** Modify or add base metals/gems, their color, etc.

### ⛏️ Items
- **File:** `common/src/main/java/ores/mathieu/material/ItemType.java`
- **Action:** Modify or add a new dynamically generated Item.

### 🧱 Blocks
- **File:** `common/src/main/java/ores/mathieu/material/BlockType.java`
- **Action:** Add/Modify the properties of a block.

### ⚙️ Ore Generation (Base Veins)
- **File:** `common/src/main/java/ores/mathieu/config/ConfigManager.java`
- **Action:** Add/Modify the default generation area, probability, and size of veins generated by the mod. Note that these are just the default values, and ore generation remains fully customizable by the user via the configuration file.

---

## 🎨 3. Help with modifying Textures

All textures are located in: `common/src/main/resources/assets/ores/textures/`

1. **Add a unique Texture**: 
   Move or create your `*.png` file in the standard folders (e.g., `block`, `item`, `trims`, or `block/ore_overlays`). 

2. **Dynamic Textures (Coloration / Palette)**: 
   Dynamic textures are based on a master file. If you want to modify the base style of ingots, clumps, or blocks, you must modify or add a visual using the exact reference color palette:
   `common/src/main/resources/assets/ores/textures/dynamics/palettes/base_palette.png`

---

## 🏆 4. Criteria for accepting a proposal

- 🔖 **Name Format**: Always prefix your contribution or issue with informative brackets, e.g., `[Material]`, `[Texture]`, `[Block]`.
- 💻 **The Code / Image proposal**: Attach your code snippet with clear values.
- 🏷️ **The short explanatory text**: Tell me **why** this addition is cool or necessary to balance the mod.

Thank you for your valuable help in improving **ORES-CORE**! 🚀
